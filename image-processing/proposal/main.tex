\section{Problem Statement}

Ground Glass Opacities (GGOs) are faint cloudy areas seen in lung CT scans. They can be early signs of diseases such as pneumonia,
lung infections, or even cancer. Finding these areas by hand is difficult and takes a lot of time because GGOs have low contrast and
spread out gradually in the lungs. Different doctors may also interpret them differently, which can affect diagnosis accuracy.

Traditional methods do not have clear and standard ways to measure GGOs, Because of this, it is very hard to accurately track how a 
disease changes over time or to compare scans taken before and after treatment.medical images stored in DICOM format are often handled inefficiently,
windowing settings (brightness and contrast) vary between hospitals, and many steps are still done manually instead of automatically.
These issues make it difficult to interpret CT scans in a consistent and standardized way.

\section{Objectives}
\begin{itemize}
    \item Improve the quality of the chest CT scans by removing the noise, normalizing the image intensity values, and converting data into a format suitable for processing.
    \item To accurately segment lung regions using HU based thresholding, morphological operations, and edge filling techniques while removing background artifacts.
    \item To detect Ground Glass Opacity regions within the segmented lung area using HU based thresholding criteria.
    \item Test the efficiency of the proposed image processing workflow on a publicly available chest CT dataset.
    \item Evaluate the effectiveness of the proposed image processing pipeline using a publicly available chest CT dataset.
    \item Design and show a simple prototype application which focuses on clear image visualization.
\end{itemize}

\section{Dataset / Data Collection}
This project utilizes the LIDC-IDRI dataset obtained from The Cancer Imaging Archive (TCIA). 
The dataset consists of over 1,000 chest CT scans stored in DICOM format. 
The key statistics of the dataset are summarized as follows:

\begin{itemize}
    \item \textbf{Total cases:} 1,018 patients
    \item \textbf{Total lesions:} 7,371 lesions marked as ``nodule'' by at least one radiologist
    \item \textbf{Nodules $\geq 3$ mm:} 2,669 lesions identified as clinically significant nodules
\end{itemize}

\noindent\textbf{Images:} linical thoracic CT scans in DICOM format.

\noindent\textbf{Annotations:} Provided in XML files containing the coordinates of nodule boundaries (contours) and associated radiological characteristic ratings.

\noindent\textbf{Tools:} The Python library \texttt{pylidc} is used to query, process, and visualize the LIDC-IDRI dataset.

\section{Expected Outcomes }

The expected outcomes of this project include:
\begin{itemize}
    \item A Python based image processing pipeline for lung CT analysis
    \item A GUI application for loading, visualizing, and navigating CT slices
    \item Accurate lung segmentation masks
    \item Highlighted Ground Glass Opacity (GGO) regions with color overlays
    \item Quantitative outputs such as GGO count, area, and estimated volume
    \item Adjustable HU threshold controls for experimentation
    \item Side by side visualization of original and processed images
\end{itemize}

\section{Percentage of Image Processing \& Computer Vision Involvement}
Core components include DICOM loading, Hounsfield Unit extraction, lung windowing, threshold-based lung segmentation, morphological operations, 
connected component analysis, flood fill, and GGO detection using HU ranges. Vessel suppression is performed using structural morphology and region-based filtering. 
Dataset preprocessing and visualization parts of the pipeline are the only components that do not involve image processing or computer vision techniques.
